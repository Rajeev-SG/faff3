# AGENTS.md

This repository is a documentation and handoff pack for the faff3 team.

## Purpose

Keep this repo easy to review, share, and update. Prefer clarity over cleverness.

## Working rules

- Treat `README.md` as the entry point for new readers.
- Keep the main business-facing outputs in `deliverables/`.
- Keep supporting evidence in `research/`.
- Keep screenshots and validation artifacts in `output/`.
- Keep cross-repo context, such as publication notes, in `meta/`.
- When adding or moving files, update `README.md` so a new reader can still find everything quickly.

## Editing guidance

- Prefer small, reversible documentation edits.
- Preserve plain-English language for partner- and client-facing material.
- Avoid introducing technical jargon unless it is necessary and explained.
- If package names, pricing, or positioning change, keep the service catalogue and strategy memo aligned.

## Validation

- For documentation-only updates, validate by checking file structure, file references, and `git status`.
- If this repo starts holding code or automation later, add repo-specific validation instructions here.
